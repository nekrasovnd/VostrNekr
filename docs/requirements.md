# Функциональные требования: Простой калькулятор

## User Stories

### US-1: Отображение интерфейса калькулятора
Как пользователь,
Я хочу видеть интерфейс калькулятора с экраном и кнопками (цифры 0-9, операции +, -, *, /, кнопки "Очистить" (C) и "Удалить символ" (DEL)),
Чтобы я мог взаимодействовать с ним для ввода данных и получения результата.

*Критерии приемки:*
* [ ] На главной странице отображается область для вывода чисел и результата (дисплей).
* [ ] Под дисплеем расположены кликабельные кнопки.
* [ ] Кнопки включают цифры от 0 до 9.
* [ ] Кнопки включают основные арифметические операции (+, -, *, /).
* [ ] Присутствуют кнопки "C" (Очистка) и "DEL" (Удаление последнего символа).
* [ ] Кнопка "=" для вычисления результата.

### US-2: Выполнение базовых арифметических операций
Как пользователь,
Я хочу вводить числа и операторы с помощью кнопок и видеть результат вычисления на экране,
Чтобы я мог быстро выполнить математические операции.

*Критерии приемки:*
* [ ] При нажатии на кнопки цифр и операторов, они отображаются на дисплее.
* [ ] При нажатии кнопки "=", система вычисляет результат выражения, отображенного на дисплее.
* [ ] Результат операции сложения двух чисел корректен.
* [ ] Результат операции вычитания двух чисел корректен.
* [ ] Результат операции умножения двух чисел корректен.
* [ ] Результат операции деления двух чисел корректен.
* [ ] После вычисления результата, нажатие на цифру или оператор начинает новое вычисление.

### US-3: Очистка ввода и исправление ошибок
Как пользователь,
Я хочу иметь возможность очистить весь ввод или удалить последний введенный символ,
Чтобы я мог легко исправить ошибку без необходимости перезапуска приложения.

*Критерии приемки:*
* [ ] Нажатие кнопки "C" полностью очищает дисплей и сбрасывает текущее вычисление.
* [ ] Нажатие кнопки "DEL" удаляет один последний символ с дисплея.
* [ ] Если дисплей пуст, нажатие "DEL" не имеет эффекта.

### US-4: Обработка особых случаев
Как пользователь,
Я хочу, чтобы калькулятор корректно обрабатывал особые случаи, такие как деление на ноль,
Чтобы приложение не ломалось и давало понятную обратную связь.

*Критерии приемки:*
* [ ] При попытке деления на ноль на дисплее отображается понятное сообщение об ошибке (например, "Error").
* [ ] После отображения ошибки, следующее нажатие на цифру или "C" очищает дисплей и позволяет начать новое вычисление.
